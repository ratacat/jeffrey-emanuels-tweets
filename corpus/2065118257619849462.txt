What It Does

This skill turns “split up a giant file” from an aesthetic refactor into a proof-driven engineering campaign. Its central rule is isomorphism: a decomposition only counts if behavior stays the same, the public API stays the same through facades and re-exports, performance stays inside noise, and compile/build resource use is neutral or better.

It is deliberately willing to conclude “do not split this file.” That matters. The skill treats generated files, justified monoliths, FFI sandwiches, shared mutable state, macro thickets, cfg thickets, hub/barrel modules, and test monoliths as different species of problem. A line count alone is never enough.

How It Works

It starts with routing and scope control: Quick, Standard, or Exhaustive mode; a sibling workspace; tool inventory; and explicit no-surgery-until-proven rules.

Then it builds evidence. The census scripts measure file size, churn, complexity, generated-file likelihood, and severity. The taxonomy buckets B1 through B12 prevent vague “big file bad” thinking and force the agent to name the actual pathology.

Next it searches for seams empirically. It builds symbol inventories, intra-file graphs, cluster candidates, coverage maps, churn coupling, import graphs, and shared-state inventories. The AST walkers for Rust, Python, Go, TypeScript, and C++ are especially important because they ask structural questions that grep cannot answer cleanly: which top-level items reference which others, and which functions are coupled through module-level state.

After that it freezes baselines: tests, goldens, API snapshots, dependency cycles, benchmarks, and compile-resource profiles. The proof gate then checks six separate axes: tests, API surface, import cycles, benchmarks, compile resource use, and new lint suppressions. Missing proof is INCOMPLETE, not success.

The experiment registry is the skill’s reasoning ledger. Every seam hypothesis needs a pathology bucket, severity, falsifiable claim, minimal probe, expected signal, invocation, and verdict. Convergence is also machine-counted: no open experiments, no unresolved refinements, reviewed deferrals, few new findings, two quiet rounds, and a Standard/Exhaustive floor of at least 10 rounds.

Once a split is justified, the skill designs the decomposition in the native idiom of the language. Rust gets module facades and pub use discipline; Python gets package __init__ surfaces and __all__; TypeScript gets explicit barrels; Go prefers same-package file splits before package splits; C++ gets umbrella headers and ODR care.

Finally it turns the plan into Beads and, only when explicitly asked, executes one mechanical move per commit with the gate after every move.
